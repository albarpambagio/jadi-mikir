# UX Audit — Settings + Export Screens (S13-S14)

**Date:** 2026-03-30  
**Auditor:** UX Audit (Standard Depth)  
**URL:** http://localhost:5173  
**Pages Tested:** Home, Settings, Export, Topics, Topic List, Progress, Session, Session Complete, 404

---

## Summary

| Severity | Count |
|----------|-------|
| Critical | 0 |
| High | 1 |
| Medium | 2 |
| Low | 3 |

**Overall:** The Settings and Export screens (S13-S14) are functional and well-designed. Minor issues found include onboarding flow navigation issues and 404 page language inconsistency.

---

## Findings

### High

#### 1. Onboarding "Lanjut" Button Not Navigating
- **Location:** `/onboarding/subject`
- **Issue:** Clicking the "Lanjut" button on the subject selection page does not navigate to the diagnostic test page. The page stays on `/onboarding/subject` despite Matematika already being selected.
- **Impact:** Users cannot proceed through onboarding naturally - they must navigate directly to `/onboarding/diagnostic` to continue.
- **Repro:** 1. Go to `/onboarding/subject` → 2. Verify "Matematika (Aktif)" is selected → 3. Click "Lanjut" → 4. Page remains on subject selection
- **Recommendation:** Investigate the click handler on the "Lanjut" button - likely a JavaScript event binding issue.

---

### Medium

#### 2. Export Page Link from Settings Not Working
- **Location:** Settings page → "Ekspor semua data sebagai JSON" link
- **Issue:** Clicking the export/import link in the Data & Privasi section does not navigate to `/settings/export`. The link appears clickable but has no effect.
- **Impact:** Users must navigate manually to export their data.
- **Repro:** 1. Go to `/settings` → 2. Click "Ekspor semua data sebagai JSON" → 3. Page stays on Settings
- **Recommendation:** Check the Link component implementation in the Settings page - verify `href` is correctly set.

#### 3. 404 Page Uses English Instead of Indonesian
- **Location:** Any invalid route (e.g., `/nonexistent`)
- **Issue:** The 404 error page displays "404 - Not found" and "The page you're looking for doesn't exist." in English, while all other UI elements are in Indonesian.
- **Impact:** Inconsistent language experience for Indonesian learners.
- **Recommendation:** Update the 404 page text to Indonesian: "404 - Halaman tidak ditemukan" and "Halaman yang kamu cari tidak ada."

---

### Low

#### 4. Session Progress Bar Shows 0% on First Question
- **Location:** Session screen
- **Issue:** The progress bar shows 0% progress even after answering the first question correctly.
- **Impact:** Minor visual inconsistency - progress should update to reflect completion.
- **Recommendation:** Update progress bar calculation to reflect current question index / total questions.

#### 5. Diagnostic Test Stuck on Feedback Screen
- **Location:** `/onboarding/diagnostic`
- **Issue:** After answering a question and seeing feedback, clicking "Pertanyaan berikutnya" does not advance to the next question. The page remains stuck on the feedback screen.
- **Impact:** Users cannot complete the diagnostic test naturally.
- **Repro:** 1. Start diagnostic → 2. Select answer → 3. Click "Konfirmasi" → 4. See feedback → 5. Click "Pertanyaan berikutnya" → 6. Stuck on same screen
- **Recommendation:** Investigate the "Pertanyaan berikutnya" button handler in the diagnostic component.

#### 6. Topics Subject Click Doesn't Navigate
- **Location:** `/topics` page
- **Issue:** Clicking on the Matematika subject card does not navigate to the topic list. Only clicking "Ganti ke subjek ini" works.
- **Impact:** Users expect clicking the card itself to navigate.
- **Repro:** 1. Go to `/topics` → 2. Click on "Matematika" card (not the link) → 3. Nothing happens
- **Recommendation:** Make the entire subject card clickable or add a cursor:pointer style to indicate it's interactive.

---

## Positive Findings

1. **Settings Page:** Well-organized sections with clear labels in Indonesian
2. **Export Page:** Clean UI with clear export/import buttons and format explanation
3. **Topic List:** Proper locked/unlocked state indicators
4. **Session Flow:** Good feedback UI with correct/incorrect highlighting
5. **Session Complete:** Clear stats display with progress toward mastery gate
6. **Mobile Responsive:** Layout adapts well to smaller viewports
